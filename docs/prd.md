# 제품 요구사항 정의서 (PRD - Product Requirements Document)

## 1. 개요
**AI PDF Parser**는 기업 및 개인 사용자가 민감한 내용을 포함한 PDF 문서를 오프라인 상태에서 안전하게 분석하고, 필요한 데이터를 추출할 수 있도록 돕는 데스크톱 애플리케이션입니다. **2-Layer 아키텍처**와 **Vision 멀티모달 기술**을 결합하여 텍스트뿐만 아니라 표, 레이아웃 등 시각적 정보까지 완벽하게 추출하는 것을 목표로 합니다.

## 2. 주요 기능 및 사용자 시나리오

1. **지능형 멀티모달 추출 (Vision Mode)**
   - 단순히 텍스트만 읽어서는 파악하기 힘든 복잡한 표나 이미지 기반의 PDF를 처리할 때, 사용자는 'Vision' 모드를 활성화합니다. 앱은 PDF 페이지를 실시간으로 이미지로 변환하여 비전 지원 모델에 전달하고, 시각적 맥락을 반영한 정교한 JSON 데이터를 추출합니다.

2. **데이터 추출 이력 관리 (Extraction History)**
   - 사용자가 이전에 작업했던 추출 결과를 잃지 않도록 이력 사이드바를 제공합니다. 각 이력은 추출 당시 사용된 모델, 파라미터, 프롬프트 전문을 스냅샷으로 저장하여 나중에 언제든 동일한 환경을 복기할 수 있습니다.

3. **모델별 맞춤형 환경설정 (Per-Model Settings)**
   - 모델마다 최적의 `temperature`나 `max tokens` 값이 다를 수 있습니다. 사용자가 특정 모델을 선택하고 설정을 변경하면, 해당 모델 전용 설정 파일(`.json`)에 자동으로 저장되어 다음에 해당 모델을 고를 때 즉시 복원됩니다.

4. **철저한 리소스 통제 및 모니터링**
   - Windows Job Object 기술을 활용하여 서브프로세스(`llama-server`)를 강력하게 통제하며, 앱 종료 시 VRAM 점유를 즉시 해제합니다. 하단 상태바를 통해 RAM/VRAM 사용량과 추론 성능(TPS, TTFT)을 실시간으로 확인합니다.

## 3. 상세 기능 요구사항

### 3.1 하드웨어 및 엔진
- **2-Layer 아키텍처**: Tauri 백엔드와 추론 엔진(`llama-server`)의 완전 분리로 안정성 확보.
- **다중 런타임**: CPU, Vulkan, CUDA 12 환경 지원.
- **자동 최적화 (Auto NGL)**: 시스템 VRAM 용량을 감지하여 최적의 GPU 오프로드 레이어 수 자동 제안.

### 3.2 데이터 및 파일 관리
- **계층적 모델 저장**: `{root}/{owner}/{repo}/{model}.gguf` 구조로 체계적인 모델 관리.
- **표준 비전 프로젝터**: `mmproj.gguf` 네이밍 표준화를 통한 멀티모달 자동 로드.
- **데이터 영속성**: 앱 재시작 시에도 모든 설정, 프롬프트, 히스토리 데이터 복원.

### 3.3 사용자 인터페이스 (UI/UX)
- **고정형 반응형 레이아웃**: 컨텐츠 크기에 상관없이 안정적으로 유지되는 뷰포트.
- **Hugging Face 지능형 검색**: 태그 및 파일 구성을 분석하여 Vision 지원 여부를 즉각적으로 식별.
- **실시간 피드백**: 토큰 스트리밍 및 진행 상태 알림 제공.
