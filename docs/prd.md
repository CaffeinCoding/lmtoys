# 제품 요구사항 정의서 (PRD - Product Requirements Document)

## 1. 개요
**AI PDF Parser**는 기업 및 개인 사용자가 민감한 내용을 포함한 PDF 문서를 오프라인 상태에서 안전하게 분석하고, 필요한 데이터를 추출할 수 있도록 돕는 데스크톱 애플리케이션입니다. **2-Layer 아키텍처**와 **Vision 멀티모달 기술**을 결합하여 텍스트뿐만 아니라 표, 레이아웃 등 시각적 정보까지 완벽하게 추출하는 것을 목표로 합니다.

## 2. 주요 기능 및 사용자 시나리오

1.  **지능형 멀티모달 추출 (Vision Mode)**
    *   **시나리오**: 사용자가 복잡한 표나 이미지 기반의 PDF를 처리할 때 'Vision' 모드를 활성화합니다. 앱은 PDF 페이지를 고해상도 이미지로 변환하여 비전 지원 모델에 전달하고, 시각적 맥락(테두리, 배치, 색상 등)을 반영한 정교한 데이터를 추출합니다.
    *   **기능**: 페이지별 순차 추론, 이미지 해상도 조절, GBNF 문법 기반 결과 일관성 유지.

2.  **모델별 맞춤형 환경설정 (Per-Model Settings)**
    *   **시나리오**: 사용자가 가벼운 'Gemma' 모델에서는 Temperature를 0.1로, 더 큰 'Llama' 모델에서는 0.7로 설정하고 싶어 합니다. 모델을 변경할 때마다 매번 설정할 필요 없이, 앱이 각 모델 전용 설정 파일(`.json`)을 자동으로 관리하여 즉시 복원합니다.
    *   **기능**: 모델 로드 시 설정 오버라이드, 파라미터 스냅샷 자동 저장.

3.  **데이터 추출 이력 및 스냅샷 (Extraction History)**
    *   **시나리오**: 사용자가 어제 추출했던 결과의 정확도가 높았다고 판단하여 당시의 설정값을 복구하고자 합니다. 히스토리 사이드바에서 해당 항목을 클릭하면 당시 사용된 모델, 프롬프트, 파라미터 전문이 복원됩니다.
    *   **기능**: 추출 결과 및 설정 스냅샷 저장, 히스토리 관리.

4.  **철저한 리소스 통제 및 실시간 모니터링**
    *   **시나리오**: 사용자가 고사양 GPU 모델을 구동하다가 앱을 갑자기 닫아도 VRAM이 즉시 해제되어야 합니다. 또한 하단 상태바를 통해 실시간으로 VRAM 점유율을 확인하며 무리한 오프로딩을 방지합니다.
    *   **기능**: Windows Job Object 연동, RAM/VRAM 실시간 쿼리, 추론 성능 지표(TPS, TTFT) 피드백.

## 3. 상세 기능 요구사항

### 3.1 하드웨어 및 엔진
-   **다중 런타임**: 사용자의 GPU 유무에 따라 CPU, Vulkan, CUDA 12 바이너리를 선택 구동할 수 있어야 합니다.
-   **자동 최적화 (Auto NGL)**: 시스템 VRAM 용량을 감지하여 최적의 GPU 오프로드 레이어 수(NGL)를 자동으로 제안해야 합니다.

### 3.2 데이터 및 파일 관리
-   **계층적 모델 저장**: 다운로드된 모델은 `{root}/{owner}/{repo}/{model}.gguf` 구조로 관리되어야 하며, 재귀적 스캔을 통해 탐지되어야 합니다.
-   **표준 비전 프로젝터**: 모델 폴더 내 `mmproj.gguf` 네이밍 표준화를 통해 별도의 설정 없이도 Vision 기능이 즉각 활성화되어야 합니다.

### 3.3 사용자 인터페이스 (UI/UX)
-   **반응형 레이아웃**: PDF 뷰어와 설정 패널이 독립적으로 스크롤되며 쾌적한 뷰포트를 유지해야 합니다.
-   **스트리밍 피드백**: 로컬 서버의 stdout/stderr 로그와 토큰 생성 과정을 실시간으로 확인할 수 있어야 합니다.

## 4. 비기능 요구사항
-   **보안**: 모든 추론 프로세스는 로컬 서버 내에서 완결되어야 하며, 명시적인 사용자 요청 없이 외부 네트워크로 데이터를 전송해서는 안 됩니다.
-   **안정성**: `llama-server` 크래시 발생 시 이를 즉각 감지하고 사용자에게 알림을 제공해야 합니다.
