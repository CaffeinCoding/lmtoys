# 제품 요구사항 정의서 (PRD - Product Requirements Document)

## 1. 개요
**AI PDF Parser**는 기업 및 개인 사용자가 민감한 내용을 포함한 PDF 문서를 오프라인 상태에서 안전하게 분석하고, 필요한 데이터를 추출할 수 있도록 돕는 데스크톱 애플리케이션입니다. 외부 서버로 데이터를 전송하지 않고 로컬 하드웨어를 활용하여 대형 언어 모델(LLM)을 구동함으로써 보안 이슈를 원천 차단하는 것을 목표로 합니다.

## 2. 사용자 시나리오 (User Scenarios)

1. **오프라인 보안 환경에서의 데이터 추출**
   - 금융, 의료, 법률 분야의 종사자가 외부 인터넷 접속이 제한되거나 클라우드 전송이 금지된 환경에서 방대한 PDF 자료를 요약하거나 중요 엔티티를 JSON 형태로 추출해야 합니다. 사용자는 앱 내장 엔진(Built-in)을 통해 다운로드해둔 GGUF 모델을 선택하여 문서를 처리합니다.

2. **클라우드 API를 활용한 고성능 처리**
   - 보안 이슈가 적은 문서의 경우, 사용자는 OpenAI, Claude, Gemini 등의 클라우드 API를 연결하여 보다 빠르고 정교한 텍스트 파싱 결과를 얻고자 합니다.

3. **리소스 모니터링 및 최적화**
   - 개인 노트북이나 로컬 워크스테이션에서 모델을 구동할 때, 사용자는 시스템 메모리(RAM)와 그래픽 메모리(VRAM)가 꽉 차지 않게 모니터링하기를 원하며, 초당 토큰 처리 속도(TPS)를 실시간으로 확인하여 모델의 성능을 체감하고자 합니다.

## 3. 핵심 요구사항 (Key Requirements)

### 3.1 기능적 요구사항 (Functional Requirements)
- **PDF 텍스트 추출**: PDF 파일 경로를 입력받아 문서 내의 텍스트를 정확하게 추출해야 합니다.
- **다중 LLM 환경 지원**:
  - 클라우드 API (OpenAI, Gemini, Claude) 지원
  - 외부 로컬 서버 (Ollama, LM Studio) 지원
  - **내장 로컬 엔진(Built-in)**: 앱 자체적으로 `llama_cpp`를 내장하여 백엔드에서 GGUF 모델을 구동
- **모델 관리 기능**: 
  - Hugging Face와 같은 외부 소스에서 GGUF 모델을 다운로드 (인증 토큰 지원)
  - 로컬에 다운로드된 모델 목록 조회, 메타데이터 읽기, 모델 삭제 기능
- **추론 파라미터 조절**:
  - Temperature, Max Tokens, Top-P, Repeat Penalty 조절 UI
  - 시스템 프롬프트 및 사용자 정의 JSON 형식 포맷 수정 인터페이스 제공
- **하드웨어 가속**: 빌드 타임에 CPU, CUDA, Vulkan 등 다중 런타임을 구성하여, 사용자 환경에 맞게 GPU 가속 여부를 결정할 수 있어야 합니다. (nGpuLayers 조절)
- **시스템 상태 실시간 모니터링**: 
  - 앱 전역에서 VRAM, RAM 사용량 표시
  - 실시간 스트리밍 답변과 함께 TPS 및 TTFT 성능 측정 표시

### 3.2 비기능적 요구사항 (Non-Functional Requirements)
- **보안성**: 로컬 모델 사용 시, 사용자의 어떠한 문서 데이터도 외부 네트워크로 유출되지 않아야 합니다.
- **반응성**: 무거운 모델 구동 중에도 UI가 멈추지 않아야 하며, 토큰 스트리밍 기술을 적용해 사용자가 즉각적인 피드백을 받을 수 있어야 합니다.
- **크로스 플랫폼**: Tauri 기반으로 Windows, macOS, Linux 데스크톱 환경을 지원해야 합니다.
