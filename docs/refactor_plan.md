# 프로젝트 리팩토링 및 고도화 완료 보고서

## ✅ 1. 2-Layer 아키텍처 도입 (완료)
Tauri Main 프로세스와 `llama-server` Subprocess를 분리하여 시스템의 안정성과 유연성을 확보했습니다.

- **서브 프로세스 관리**: `std::process::Command`를 통한 llama-server 제어.
- **🚨 좀비 프로세스 차단**: Windows Job Object API를 적용하여 메인 앱 종료 시 자식 프로세스 즉시 강제 종료 보장.
- **멀티 런타임**: CPU, Vulkan, CUDA 12 바이너리 선택 구동 지원.

## ✅ 2. 멀티모달 비전(Vision) 기능 구현 (완료)
시각 정보 기반의 정교한 데이터 추출 파이프라인을 구축했습니다.

- **표준화된 탐지**: 모델 폴더 내 `mmproj.gguf` 자동 식별 및 로드.
- **이미지 스트림**: PDF 페이지를 고해상도 이미지로 실시간 렌더링하여 LLM에 전달.
- **지능형 검색**: Hugging Face 검색 시 파일 구성을 분석하여 Vision 지원 모델 즉시 식별.

## ✅ 3. 데이터 영속성 및 관리 고도화 (완료)
사용자의 작업 이력과 설정을 완벽하게 보존합니다.

- **이력 관리**: 추출 당시의 모델, 파라미터, 프롬프트 스냅샷을 포함한 히스토리 사이드바 구현.
- **모델별 설정**: 각 모델마다 최적화된 파라미터를 개별 `.json` 파일로 자동 관리.
- **상태 복원**: `isInitializing` 가드 로직을 통해 앱 재시작 시 모든 환경을 100% 복원.

## ✅ 4. 계층적 모델 저장 체계 (완료)
Hugging Face의 대규모 모델 라이브러리를 효율적으로 관리합니다.

- **구조**: `{다운로드 경로}/{Owner}/{Repo}/{Model}.gguf`
- **재귀적 스캔**: 깊은 폴더 구조 내의 모든 GGUF 모델을 자동으로 탐지하여 리스팅.
