# lmtoys 사용자 가이드 (User Guide)

**lmtoys**(Local Multimodal Toys)는 민감한 데이터를 포함한 PDF 문서를 로컬 환경에서 안전하게 분석하고 지능적으로 데이터를 추출할 수 있도록 돕는 데스크톱 애플리케이션입니다. 이 가이드는 앱의 주요 기능과 효율적인 사용 방법을 설명합니다.

---

## 1. 시작하기 (Getting Started)

### 사전 요구 사항

- **OS**: Windows 10/11
- **하드웨어**:
    - 기본 실행: CPU 및 8GB 이상의 RAM
    - 가속 실행: NVIDIA GPU (CUDA 지원) 또는 Vulkan 지원 GPU 권장
- **네트워크**: 모델 다운로드 시 인터넷 연결 필요 (추론은 오프라인 가능)

---

## 2. 모델 설정 및 관리 (Settings)

앱을 처음 실행하면 좌측 메뉴의 **Settings** 탭에서 모델을 구성해야 합니다.

### 2.1 클라우드 모델 (Cloud Models)

- **OpenAI, Google Gemini, Anthropic Claude**의 API 키를 입력하여 강력한 클라우드 모델을 사용할 수 있습니다.
- 설정된 키는 로컬 저장소에 안전하게 저장됩니다.

### 2.2 로컬 모델 (Local Models)

- **Ollama / LM Studio**: 이미 실행 중인 외부 로컬 서버가 있다면 URL을 입력하여 연동할 수 있습니다.
- **내장 엔진 (llama-server)**: 앱 내에서 직접 GGUF 모델을 다운로드하고 구동할 수 있습니다.
    - **Hugging Face 검색**: 원하는 모델(예: `Llama-3`, `Gemma-4`, `Qwen`)을 검색하고 다운로드합니다.
    - **Vision 지원**: 눈 아이콘(👁️)이 표시된 모델은 이미지 분석(Vision Mode)이 가능합니다.
        - 'mmproj-\*.gguf' 파일을 다운로드 받아야 합니다.
    - **GPU 가속**: 설정에서 `CUDA` 또는 `Vulkan`을 선택하여 추론 속도를 높일 수 있습니다.

---

## 3. PDF 데이터 추출 (Home)

**Home** 화면은 PDF를 분석하고 데이터를 추출하는 핵심 공간입니다.

### 3.1 파일 업로드 및 분석

1. 분석할 PDF 파일을 드래그 앤 드롭하거나 선택하여 업로드합니다.
2. 사이드바에서 사용할 **모델**을 선택합니다.

### 3.2 추출 모드 선택

- **Text Mode**: PDF의 텍스트 스트림을 직접 추출하여 전달합니다. 속도가 빠르고 일반적인 문서에 적합합니다.
- **Vision Mode (강력 권장)**:
    - PDF 페이지를 이미지로 변환하여 모델에게 전달합니다.
    - **표(Table), 복잡한 레이아웃, 손글씨** 등이 포함된 문서에서 압도적인 성능을 발휘합니다.
    - Vision 전용 모델(`mmproj` 포함)이 로드되어 있어야 활성화됩니다.

### 3.3 프롬프트 및 파라미터 설정

- **System Prompt**: 모델의 역할을 정의합니다. (예: "너는 회계 전문가야.")
- **User Instruction**: 추출하고 싶은 데이터 형식을 지정합니다. (예: "날짜, 금액, 항목을 JSON 형태로 추출해줘.")
- **Temperature**: 값이 낮을수록 일관된 결과를, 높을수록 창의적인 결과를 출력합니다.

---

## 4. 데이터 확인 및 내보내기 (Data Viewer)

추출된 결과는 **Data Viewer** 탭에서 관리할 수 있습니다.

- **실시간 결과 확인**: 모델이 생성한 데이터를 즉시 확인합니다.
- **이력 관리**: 과거에 추출했던 데이터와 당시의 설정값(모델, 프롬프트 등)을 함께 확인할 수 있습니다.
- **Excel 내보내기**: 추출된 데이터를 엑셀 파일(`.xlsx`)로 저장하여 업무에 즉시 활용할 수 있습니다.

---

## 5. 시스템 모니터링 및 최적화

### 상태바 (Global Status Bar)

화면 하단에서 실시간 시스템 상태를 확인할 수 있습니다.

- **RAM / VRAM**: 현재 메모리 점유율을 모니터링하여 시스템 과부하를 방지합니다.
- **추론 지표**: TPS(초당 토큰 수), TTFT(첫 토큰 응답 시간)를 통해 모델의 성능을 체크합니다.

### 최적화 팁

- **Auto NGL**: VRAM 용량에 맞춰 GPU 레이어 수(NGL)를 조정하면 속도를 최적화할 수 있습니다.
- **Vision Resolution**: 설정에서 Vision 해상도를 조정하여 정확도와 VRAM 사용량 사이의 균형을 맞출 수 있습니다.

---

## 6. 문제 해결 (Troubleshooting)

- **서버 연결 실패**: 내장 엔진 사용 시 포트 충돌이 있는지 확인하거나 다른 포트로 변경해 보세요.
- **VRAM 부족**: 모델 크기를 줄이거나(`Q4_K_M` 권장), NGL 값을 낮추어 설정을 변경해 보세요.
- **추출 결과 부정확**: 'Vision Mode'를 활성화하고, 프롬프트에 추출 요구사항을 더 구체적으로 명시해 보세요.
